Predicting Gaming Hours
with Linear Regression

Using the Steam User Behavior Dataset to predict total play time based on a user's game library size.

Problem Statement

Can we predict how many hours a user will play?

Steam is the world's largest digital distribution platform for PC gaming, with millions of active users and thousands of available titles. Every day, users purchase games and accumulate hours of playtime across their libraries.

Understanding gaming behavior patterns is valuable for game developers, platform designers, and recommendation systems. One key question is: can we predict a user's total play time based on the number of games they own?

We apply simple linear regression to model this relationship, using real-world data from the Steam platform.

Why does this matter?

Predicting user engagement helps platforms personalize recommendations, optimize server resources, and identify high-value users. Linear regression gives us a simple yet interpretable baseline model for this task.

Dataset

Steam User Behavior Dataset

The dataset contains 200,000 rows of user-game interactions collected from the Steam platform. Each row represents a specific relationship between a user and a game.

There are two types of records:

  • Purchase — indicates the user owns the game
  • Play — indicates play time logged for that game

By aggregating these records at the user level, we can derive features like total games owned and total hours played.

Variables

Variables Used in the Model

X
Independent Variable
Number of Games Owned

Total count of unique games purchased by the user on Steam.

Y
Dependent Variable
Total Hours Played

Sum of all play hours logged by the user across every game in their library.

Methodology

Data Preparation Steps

1
Filter Purchase Records

Select only rows where the action is "purchase" and count the number of unique games per user.

2
Filter Play Records

Select only rows where the action is "play" and sum the total hours played per user.

3
Merge into User-Level Dataset

Join the two aggregated tables on User ID, creating one row per user with games owned and total hours.

4
Train/Test Split (80/20)

Split the dataset into 80% training data and 20% testing data to evaluate model performance.

Visualization

Regression Graph

The scatter plot shows individual users as data points, with the number of games owned on the X-axis and total hours played on the Y-axis. The red regression line represents the model's best fit — it captures the general trend that users who own more games tend to accumulate more play hours.

What do these numbers mean?

Slope: For each additional game a user owns, the model predicts an increase of {{ slope }} hours in total play time.

Intercept: A user with zero games would have an estimated base play time of {{ intercept }} hours (theoretical baseline).

R² Score: The coefficient of determination indicates how well the model explains the variance in the data. A value of {{ r2_score }} means the model captures that proportion of the variability in hours played based on games owned.
