K-Nearest Neighbors
Basic Concepts

Understanding how KNN classifies data by finding the most similar examples in the training set.

Definition

What is K-Nearest Neighbors?

K-Nearest Neighbors (KNN) is a supervised machine learning algorithm used for both classification and regression tasks. It is one of the simplest and most intuitive algorithms in machine learning.

KNN is a lazy learner (also called instance-based learning), meaning it does not build an explicit model during training. Instead, it stores the entire training dataset and makes predictions at query time by looking at the closest data points.

The core idea is simple: "Tell me who your neighbors are, and I will tell you who you are." A new data point is classified based on the majority vote of its K nearest neighbors in the feature space.

Algorithm

How KNN Works

A step-by-step overview of the classification process.

Step 1
Choose K

Select the number of neighbors (K) to consider for the vote.

Step 2
Calculate Distance

Compute the distance between the new point and all training points.

Step 3
Find Neighbors

Select the K training points closest to the new data point.

Step 4
Majority Vote

Assign the class that appears most frequently among the K neighbors.

d(p, q) = √(Σ(pi - qi)²)

Characteristics

Main Characteristics

Non-parametric
No Assumptions

KNN makes no assumptions about the underlying data distribution. It works with any shape of decision boundary, unlike logistic regression which assumes linearity.

Lazy Learner
No Training Phase

There is no explicit training step. The algorithm simply memorizes the dataset. All computation happens during prediction, making training instant but prediction slower.

Instance-Based
Uses Raw Data

Predictions are based directly on the stored training instances rather than a learned set of parameters or weights.

Distance-Based
Feature Scaling Required

Since KNN relies on distance calculations, all features must be scaled to the same range. Otherwise, features with larger magnitudes dominate the distance computation.

Hyperparameter

Choosing the Value of K

The value of K is the most important hyperparameter in KNN. It determines how many neighbors influence the classification decision.

Key guidelines for choosing K:

  • Small K (e.g., 1-3): More sensitive to noise and outliers, can lead to overfitting. Creates complex, jagged decision boundaries.
  • Large K (e.g., 15-50): Smoother decision boundaries, more robust to noise, but may underfit and blur class boundaries.
  • Odd K: Preferred for binary classification to avoid tie-breaking situations.
  • Common rule of thumb: Start with K = √N where N is the number of training samples.

In our application, we use K = 7, which provides a good balance between capturing local patterns and avoiding noise sensitivity.

Distance

Distance Metrics

Most Common
Euclidean Distance

The straight-line distance between two points in feature space. Works well when features have similar scales.

√(Σ(xi - yi)²)
Alternative
Manhattan Distance

The sum of absolute differences along each dimension. Better suited for high-dimensional or sparse data.

Σ|xi - yi|
Trade-offs

Advantages and Limitations

Advantages
  • Simple to understand and implement
  • No training phase needed
  • Naturally handles multiclass problems
  • No assumptions about data distribution
  • Can capture complex non-linear boundaries
  • Easy to add new data (no retraining)
Limitations
  • Slow predictions on large datasets
  • Sensitive to irrelevant features
  • Requires feature scaling
  • High memory usage (stores all data)
  • Suffers from the curse of dimensionality
  • Choosing the right K can be challenging
Applications

Types of Classification Problems

KNN works well for classification problems where:

  • Decision boundaries are irregular — KNN can model complex, non-linear boundaries naturally.
  • The dataset is small to medium-sized — KNN's prediction time scales linearly with dataset size.
  • Classes overlap partially — the probabilistic voting mechanism handles overlapping classes gracefully.
  • Interpretability matters — you can always explain predictions by showing the neighbors.

Common application domains include recommendation systems, pattern recognition, medical diagnosis, text categorization, and image classification.

Real World

Real-World Use Example: ESRB Rating Prediction

In our application, we use KNN to predict the ESRB rating (E, E10+, T, M) of a video game based on its characteristics such as genre, sales figures, platform, and critical reception.

Why KNN for this problem? ESRB ratings are influenced by a complex combination of factors that do not follow simple linear patterns. A violent shooter game (rated M) has very different characteristics from a family-friendly sports game (rated E). KNN can capture these non-linear patterns by finding the most similar games in the dataset.

The model analyzes 6,943 video games to learn which combinations of features are associated with each rating category, then classifies new games by comparing them to the 7 most similar games it has seen.
