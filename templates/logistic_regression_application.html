Console Wars
Platform Prediction

Using Logistic Regression to predict whether a video game belongs to PlayStation, Xbox, or Nintendo based on its characteristics.

Problem Statement

Can we predict which platform a game belongs to?

The video game industry has been shaped by the fierce competition between three major platform families: PlayStation, Xbox, and Nintendo. Each platform has distinct characteristics in terms of the types of games available, regional popularity, and audience preferences.

This application uses Logistic Regression to classify video games into their platform family based on features like sales distribution across regions, critical reception, and game genre.

By analyzing patterns in over 6,000 games, the model learns which combinations of features are most associated with each platform family.

Why does this matter?

Understanding platform-specific game characteristics helps publishers make strategic decisions about game development and distribution. It also reveals how regional preferences and game genres influence the gaming ecosystem.

Dataset

Video Games Sales Dataset

The dataset contains 16,719 video games with sales data as of December 2016, sourced from VGChartz. It includes games across 31 platforms spanning multiple console generations.

Methodology

Data Preparation Steps

1
Map Platforms to Families

Group 19 individual platforms into 3 families: PlayStation, Xbox, and Nintendo. Discard platforms that do not belong to these families (PC, retro consoles, etc.).

2
Clean Missing Data

Convert User_Score from string to numeric (removing "tbd" entries). Drop rows with missing Critic_Score, User_Score, or Genre values.

3
Encode Categorical Variables

Apply one-hot encoding to the Genre column, creating 12 binary features (one per genre). Encode the target variable using LabelEncoder.

4
Scale Numeric Features

Apply StandardScaler to normalize all features to zero mean and unit variance, ensuring the model treats all features equally.

5
Train/Test Split (80/20)

Split the dataset into 80% training and 20% testing with stratification to preserve class proportions.

Training

Model Training Process

The model uses scikit-learn's LogisticRegression with the following configuration:

  • max_iter=1000 — Maximum iterations for the solver to converge
  • class_weight='balanced' — Automatically adjusts weights to handle the class imbalance (Xbox has fewer samples than PlayStation and Nintendo)
  • random_state=42 — Ensures reproducible results

The model learns a set of coefficients (weights) for each feature and each class. During prediction, it computes probabilities for all three classes and assigns the game to the class with the highest probability.

Evaluation

Confusion Matrix

How to Read the Confusion Matrix

Each row represents the actual class and each column represents the predicted class. The diagonal values (top-left to bottom-right) show correct predictions.

Off-diagonal values show misclassifications. For example, if the Nintendo-Xbox cell has a high value, it means many Nintendo games were incorrectly classified as Xbox games.

ROC Analysis

ROC Curves and AUC

Understanding ROC and AUC

ROC (Receiver Operating Characteristic) curve plots the True Positive Rate vs False Positive Rate at various classification thresholds.

AUC (Area Under the Curve) summarizes the ROC curve into a single number between 0 and 1. An AUC of 1.0 means perfect classification, while 0.5 means no better than random guessing.

Since this is a multiclass problem, we use the One-vs-Rest (OvR) approach: one ROC curve is computed per class, treating it as a binary problem against all other classes.

Analysis

Interpretation of Results

Regional sales patterns are the strongest predictors. Japanese sales (JP_Sales) are a strong indicator for Nintendo games, as Nintendo has historically dominated the Japanese market. North American sales tend to be higher for Xbox titles, reflecting the brand's popularity in the US market.

Genre plays a meaningful role. Certain genres like Platform and Role-Playing games are more common on Nintendo consoles, while Shooter and Sports games have stronger associations with PlayStation and Xbox.

The model handles class imbalance well. Despite Xbox having fewer games in the dataset, the balanced class weights ensure the model does not simply default to predicting the majority class.

Limitations: The model uses a linear decision boundary, which may not capture complex non-linear relationships. Some games are genuinely multi-platform, making platform prediction inherently challenging. The dataset only covers sales up to 2016, so recent trends are not reflected.
