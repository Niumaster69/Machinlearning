Supervised Learning — Linear Regression

Basic Concepts

Understanding the fundamentals of linear regression, one of the most widely used algorithms in machine learning for predicting continuous values.

Introduction

What is Linear Regression?

A foundational supervised learning algorithm

Linear regression is a supervised machine learning algorithm used to predict a continuous numerical value based on one or more input features. It works by finding the best-fit straight line through a set of data points.

The goal is simple: given a set of observations, the algorithm learns the relationship between input variables and the output, and uses that relationship to make predictions on new, unseen data.

It is called "linear" because it assumes a straight-line relationship between the variables. Despite its simplicity, linear regression is one of the most powerful and widely used techniques in data science, statistics, and machine learning.

Supervised

Learns from Labels

The model trains on labeled data where both inputs and outputs are known.

Predictive

Continuous Output

Predicts numerical values like price, temperature, or hours played.

Interpretable

Easy to Understand

Results are transparent: you can see exactly how each variable affects the prediction.

Core Concepts

Independent and Dependent Variables

The two building blocks of any regression model

X

Independent Variable

Also called the predictor or feature. This is the input that we use to make a prediction. It is the variable we control or observe.

Example: The number of games a player owns in their library.

Y

Dependent Variable

Also called the response or target. This is the output we want to predict. Its value depends on the independent variable.

Example: The total hours played across all games.

Data Preparation

Training Data

How the model learns from examples

Training data is a collection of labeled examples that the model uses to learn the relationship between the input features and the output. Each example consists of an input value (X) and its corresponding known output value (Y).

The quality and quantity of training data directly impact how well the model performs. More representative data leads to better predictions, while biased or insufficient data can produce unreliable results.

Why Training Data Matters

  • The model identifies patterns by studying many input-output pairs
  • Each data point helps the model adjust its internal parameters
  • Without enough data, the model cannot generalize to new cases
  • Labeled data means every input has a known correct answer

Train / Test Split

The dataset is typically divided into two parts:

Training set (70-80%) — used to teach the model
Test set (20-30%) — used to evaluate accuracy

This split ensures the model is tested on data it has never seen before, giving an honest measure of its performance.

The Model

The Regression Line

Finding the line that best fits the data

y = mx + b
The Linear Regression Equation

The regression line is a straight line drawn through the data points that best represents the overall trend. It is defined by the equation y = mx + b, where:

  • y — the predicted value (dependent variable)
  • m — the slope of the line (rate of change)
  • x — the input value (independent variable)
  • b — the y-intercept (value of y when x = 0)

Best Fit = Minimum Error

The "best fit" line is found using a method called Ordinary Least Squares (OLS). This method minimizes the sum of the squared differences between the actual data points and the values predicted by the line.

# Conceptually, the algorithm minimizes:
Error = Sum of (actual_y - predicted_y)**2

# For each data point, the vertical distance
# between the point and the line is the "residual".
# Squaring prevents negatives from cancelling out.
# The line with the smallest total error wins.

In other words, the algorithm tries every possible line and picks the one where the total prediction error is as small as possible.

Parameters

Slope and Intercept

Understanding the two parameters that define the line

m

Slope

The slope represents the rate of change. It tells you how much the predicted output (Y) changes for every one-unit increase in the input (X).

Example: If m = 11.5, it means that for each additional game a player owns, they are predicted to play approximately 11.5 more hours in total.

b

Intercept

The intercept is the starting point of the line. It represents the predicted value of Y when X equals zero.

Example: If b = 15, it suggests that even a player with zero games owned would have a baseline of about 15 hours (perhaps from free-to-play games or demos).

Interpreting Together

Combining both parameters gives the full prediction formula. With the values above:

# Full prediction formula
hours_played = 11.5 * games_owned + 15

# Example: a player with 40 games
hours_played = 11.5 * 40 + 15 = 475 hours

The slope tells you the direction and strength of the relationship, while the intercept anchors the line on the Y-axis. Together, they fully describe the linear model.

Real-World Example

Predicting Gaming Hours Based on Library Size

Putting all concepts together in practice

Imagine a gaming platform wants to estimate how many hours a new user will spend playing games, based solely on the number of games in their library. By collecting data from thousands of existing users, they can train a linear regression model:

  1. Collect data — Gather the number of games owned (X) and total hours played (Y) from existing users.
  2. Split the data — Use 80% for training and 20% for testing.
  3. Train the model — The algorithm finds the best values of m (slope) and b (intercept).
  4. Evaluate — Test the model on unseen data to check its accuracy.
  5. Predict — Use the trained model to estimate hours for new users.
