Logistic Regression
Basic Concepts

Understanding the fundamentals of logistic regression and how it differs from linear regression for classification problems.

Definition

What is Logistic Regression?

Logistic Regression is a supervised machine learning algorithm used for classification tasks. Despite its name containing "regression," it is not used to predict continuous values. Instead, it predicts the probability that an instance belongs to a particular class or category.

The algorithm uses the sigmoid function (also called the logistic function) to map any real-valued number into a value between 0 and 1, which can be interpreted as a probability.

Logistic regression is widely used in fields like medicine (disease diagnosis), finance (credit risk), marketing (customer churn), and the gaming industry (predicting platform preferences).

σ(z) = 1 / (1 + e-z)
The Sigmoid Function — maps any input to a probability between 0 and 1
Comparison

Regression vs Classification

Understanding the key difference between predicting values and predicting categories.

Regression
Continuous Output

Predicts a numeric value on a continuous scale.

  • House price prediction
  • Temperature forecasting
  • Hours played estimation

Example: "This game will sell 2.5 million copies"

Classification
Discrete Output

Predicts a category or class label from a finite set.

  • Spam vs not spam
  • Disease positive or negative
  • Platform family prediction

Example: "This game belongs to PlayStation"

Types

Binary and Multiclass Classification

Binary
Two Classes

The target variable has exactly two possible outcomes. This is the most common form of logistic regression.

  • Yes / No
  • Spam / Not Spam
  • Pass / Fail
Multiclass
Three or More Classes

The target variable has three or more categories. Uses techniques like One-vs-Rest (OvR) or multinomial logistic regression.

  • PlayStation / Xbox / Nintendo
  • Cat / Dog / Bird
  • Low / Medium / High risk

In our Console Wars application, we use multiclass classification with three classes (PlayStation, Xbox, Nintendo). The logistic regression model uses the multinomial approach, computing the probability of each class simultaneously.

Variables

Target Variable and Features

y — Target Variable (Class Label)

The variable we want to predict. In classification, it is also called the class label. It represents the category an observation belongs to.

Console Wars example: Platform_Family (PlayStation, Xbox, or Nintendo)

X — Features (Predictors)

The input variables used to make predictions. These are the characteristics or attributes of each observation that the model learns from.

Console Wars example: Genre, NA Sales, EU Sales, JP Sales, Critic Score, User Score

Core Mechanism

Probability Estimation

Unlike linear regression which outputs a continuous value, logistic regression outputs a probability between 0 and 1 for each class.

The model computes a linear combination of the input features (z = w·x + b), then passes the result through the sigmoid function to transform it into a probability.

For multiclass problems, the model uses the softmax function which generalizes the sigmoid to output a probability distribution across all classes. The sum of all class probabilities equals 1.

Step 1

Compute linear combination
z = w·X + b

Step 2

Apply sigmoid/softmax
p = σ(z)

Step 3

Assign class with highest probability

Geometry

Decision Boundary

The decision boundary is the line (or hyperplane in higher dimensions) that separates the feature space into regions, each corresponding to a different class.

Logistic regression creates a linear decision boundary. In 2D feature space, this is a straight line. In higher dimensions, it becomes a hyperplane. Points on one side of the boundary are classified as one class, and points on the other side as another class.

For multiclass problems, there are multiple decision boundaries — one separating each pair of classes. The model divides the feature space into regions where each class has the highest predicted probability.

Configuration

Classification Threshold

The classification threshold (also called the decision threshold) is the probability value above which the model assigns an observation to the positive class. By default, this is set to 0.5 for binary classification.

If the predicted probability is ≥ 0.5, the instance is classified as the positive class. If it is < 0.5, it is classified as the negative class.

Adjusting the threshold changes the trade-off between precision and recall. A lower threshold increases recall (catches more positives) but decreases precision (more false positives). A higher threshold does the opposite.

Low Threshold (0.3)

Higher recall, lower precision. Fewer missed positives.

Default (0.5)

Balanced trade-off between precision and recall.

High Threshold (0.7)

Higher precision, lower recall. More confident predictions.

Real World

Real-World Use Example: Console Wars

In our Console Wars application, we use logistic regression to predict which platform family (PlayStation, Xbox, or Nintendo) a video game most likely belongs to, based on its characteristics.

Why this matters: Understanding which platforms attract specific types of games helps publishers make strategic decisions about where to release their titles. A game with high Japanese sales and sports genre is more likely to be a Nintendo title, while a shooter with high North American sales might lean toward Xbox.

The model learns patterns from over 6,000 video games in the dataset, analyzing features like regional sales distribution, critic and user scores, and game genre to make its predictions.

This is a multiclass classification problem with three classes, making it a great example of how logistic regression extends beyond simple binary classification.
